🎯 curl로 Post Service 테스트하는 완벽한 가이드
=============================================

📌 주의사항: 
- 일반 터미널/쉘에서는 'curl' 명령어 사용
- PowerShell에서는 'curl'이 'Invoke-WebRequest'의 별칭이므로 'curl.exe' 사용
- 또는 PowerShell에서 'curl.exe' 대신 'curl.exe' 사용

🚀 기본 API 테스트
==================

1. 헬스체크
-----------
curl http://localhost:5000/health

응답 예시:
{
  "service": "Post Service API",
  "status": "healthy",
  "version": "1.0.0"
}

2. 카테고리 목록 조회
--------------------
curl http://localhost:5000/api/v1/categories

응답 예시:
{
  "data": [
    {
      "id": "781f7e257eae4a4d8041e0f62ee8810b",
      "name": "일반",
      "description": "일반적인 게시글",
      "created_at": "2025-08-21T06:01:23.367698"
    },
    {
      "id": "192443a9ab6a4fb2bb5c42ad18b11f13",
      "name": "공지사항",
      "description": "중요한 공지사항",
      "created_at": "2025-08-21T06:01:23.380474"
    }
    // ... 더 많은 카테고리들
  ],
  "message": "Success",
  "success": true
}

3. 게시글 목록 조회
------------------
curl http://localhost:5000/api/v1/posts

응답 예시:
{
  "data": [],
  "message": "Success",
  "meta": {
    "page": 1,
    "pages": 0,
    "per_page": 10,
    "total": 0
  },
  "success": true
}

4. 게시글 목록 (페이지네이션)
----------------------------
curl "http://localhost:5000/api/v1/posts?page=1&per_page=5"

5. 게시글 검색
--------------
curl "http://localhost:5000/api/v1/posts?q=테스트"

6. 카테고리별 게시글 조회
------------------------
curl "http://localhost:5000/api/v1/posts?category_id=781f7e257eae4a4d8041e0f62ee8810b"

7. 정렬 방식별 게시글 조회
-------------------------
# 최신순 (기본값)
curl "http://localhost:5000/api/v1/posts?sort=latest"

# 인기순
curl "http://localhost:5000/api/v1/posts?sort=popular"

🌐 웹페이지 테스트
==================

8. 메인 페이지 로드
------------------
curl http://localhost:5000/

9. 글쓰기 페이지 로드
--------------------
curl http://localhost:5000/write

10. 게시글 상세 페이지 로드
-------------------------
curl http://localhost:5000/post

11. 게시글 수정 페이지 로드
-------------------------
curl http://localhost:5000/edit

📝 게시글 CRUD 테스트
====================

12. 새 게시글 생성
-----------------
curl -X POST ^
  -H "Content-Type: application/json" ^
  -d "{\"title\":\"테스트 게시글\",\"content\":\"테스트 내용입니다.\",\"author_id\":\"test_user_001\",\"category_id\":\"781f7e257eae4a4d8041e0f62ee8810b\"}" ^
  http://localhost:5000/api/v1/posts

13. 게시글 수정 (PUT)
--------------------
curl -X PUT ^
  -H "Content-Type: application/json" ^
  -d "{\"title\":\"수정된 제목\",\"content\":\"수정된 내용\",\"author_id\":\"test_user_001\"}" ^
  http://localhost:5000/api/v1/posts/{POST_ID}

14. 게시글 부분 수정 (PATCH)
---------------------------
curl -X PATCH ^
  -H "Content-Type: application/json" ^
  -d "{\"title\":\"부분 수정된 제목\"}" ^
  http://localhost:5000/api/v1/posts/{POST_ID}

15. 게시글 삭제
--------------
curl -X DELETE http://localhost:5000/api/v1/posts/{POST_ID}

16. 게시글 상세 조회
------------------
curl http://localhost:5000/api/v1/posts/{POST_ID}

💖 반응(좋아요) 테스트
======================

17. 게시글 좋아요
----------------
curl -X POST ^
  -H "Content-Type: application/json" ^
  -d "{\"user_id\":\"test_user_001\"}" ^
  http://localhost:5000/api/v1/posts/{POST_ID}/like

19. 특정 카테고리 조회

  http://localhost:5000/api/v1/tags

🔍 고급 curl 옵션 테스트
========================

22. 상세 정보와 함께 확인 (헤더 포함)
-----------------------------------
curl -v http://localhost:5000/health

23. 응답 헤더만 확인
-------------------
curl -I http://localhost:5000/health

24. 응답 시간 측정
-----------------
curl -w "응답시간: %{time_total}초\n" http://localhost:5000/health

25. HTTP 상태 코드만 확인
------------------------
curl -s -o /dev/null -w "%{http_code}" http://localhost:5000/health

26. 특정 헤더 추가
-----------------
curl -H "User-Agent: PostService-Test" http://localhost:5000/health

27. 쿠키 전송
-------------
curl -b "session=test123" http://localhost:5000/health

28. 리다이렉트 따라가기
----------------------
curl -L http://localhost:5000/health

29. 타임아웃 설정
----------------
curl --max-time 5 http://localhost:5000/health

30. 재시도 설정
--------------
curl --retry 3 http://localhost:5000/health

📊 응답 데이터 처리
===================

31. JSON 응답을 예쁘게 출력 (Windows)
-----------------------------------
curl -s http://localhost:5000/api/v1/categories | python -m json.tool

32. 특정 내용만 필터링
---------------------
curl -s http://localhost:5000/api/v1/categories | findstr "name"

33. 응답을 파일로 저장
---------------------
curl -o response.json http://localhost:5000/api/v1/categories

34. 응답 헤더를 파일로 저장
-------------------------
curl -D headers.txt http://localhost:5000/api/v1/categories

🔧 문제 해결 테스트
===================

35. 연결 테스트
--------------
curl --connect-timeout 10 http://localhost:5000/health

36. SSL 검증 건너뛰기 (HTTPS인 경우)
----------------------------------
curl -k https://localhost:5000/health

37. 프록시 설정
--------------
curl -x proxy.example.com:8080 http://localhost:5000/health

38. 사용자 인증 (필요한 경우)
---------------------------
curl -u username:password http://localhost:5000/health

📋 테스트 시나리오
==================

시나리오 1: 기본 기능 테스트
----------------------------
1. curl http://localhost:5000/health
2. curl http://localhost:5000/api/v1/categories
3. curl http://localhost:5000/api/v1/posts

시나리오 2: 게시글 생명주기 테스트
--------------------------------
1. curl -X POST -H "Content-Type: application/json" -d "{\"title\":\"테스트\",\"content\":\"내용\",\"author_id\":\"test_user_001\",\"category_id\":\"781f7e257eae4a4d8041e0f62ee8810b\"}" http://localhost:5000/api/v1/posts
2. 응답에서 post_id 추출
3. curl http://localhost:5000/api/v1/posts/{POST_ID}
4. curl -X PUT -H "Content-Type: application/json" -d "{\"title\":\"수정된 제목\"}" http://localhost:5000/api/v1/posts/{POST_ID}
5. curl -X DELETE http://localhost:5000/api/v1/posts/{POST_ID}

시나리오 3: 성능 테스트
----------------------
1. curl -w "응답시간: %{time_total}초\n" http://localhost:5000/health
2. curl -w "응답시간: %{time_total}초\n" http://localhost:5000/api/v1/categories
3. curl -w "응답시간: %{time_total}초\n" http://localhost:5000/api/v1/posts

💡 유용한 팁
============

1. 일반 터미널/쉘에서는 'curl' 명령어 사용
2. PowerShell에서는 'curl.exe' 사용 (또는 'curl.exe' 사용)
3. 긴 명령어는 ^ (캐럿)으로 줄바꿈
4. JSON 데이터는 이스케이프 처리 필요
5. 응답 확인을 위해 -v 옵션 활용
6. 자동화를 위해 -s (silent) 옵션 사용

🚨 주의사항
===========

1. POST/PUT 요청 시 Content-Type 헤더 필수
2. JSON 데이터의 따옴표 이스케이프 처리
3. Windows에서 ^ 문자 사용 시 주의
4. PowerShell에서는 curl.exe 사용 권장
5. 포트 5000이 사용 가능한지 확인
6. 도커 컨테이너가 실행 중인지 확인

🎯 결론
=======

이 가이드를 통해 Post Service의 모든 기능을 curl로 테스트할 수 있습니다.
각 API 엔드포인트의 동작을 확인하고, 성능을 측정하며, 문제를 진단할 수 있습니다.

테스트 시에는 항상 도커 컨테이너가 실행 중인지 확인하고,
응답 코드와 내용을 주의 깊게 관찰하세요!
